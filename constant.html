String Constants

In Malina, there are two types of string constants: quoted strings and open strings.

Quoted strings

Malina uses double quoted and singled quote strings.

To escape double quote in a double-quoted string use two consequent double quotes ( "" ). The same approach works for single quote.

@partNum = "748-OT"
productName = "Diamond ""Quoted"" heart"
quantity = "1"
price = "248.90"
ipo.comment = 'Valentine''s day packaging.'

Escape characters

Malina allows using escape characters in the single quoted strings.

The following formats are used to insert UTF-8/16 escape character into the string:

Use case (for new line character)

decimal

hex

Escape symbol not followed by digit (decimal or hex)

$10

$%A

Escape symbol followed by a digit (decimal or hex). Max length is 6 symbols including $ or $%.

$(10), $00010

$(%A), $%000A

Example

‘Line 1$10Line2’, ‘Line3$(10)Line4’

‘Line 1$%A Line2’, ‘Line3$(%A)Line4’

 

Note: Because $ is used for escape characters, it should be escaped if it actually means a dollar sign. Dollar sign can be escaped as "$$" or "\$".

String Interpolation

String interpolation allows you to insert a value of an alias inside a string. The interpolation has the following syntax. $(alias_name). The parentheses are optional if interpolation is followed by space or another symbol that can’t be used in the alias name. Examples:

line1 = 'The customer name is $Customer.NJ.Newark' line2 = 'The customer\'s phone number is $(Customer.Phone.AreaCode)-$(Customer.Phone.LocalNumber)-$Customer.Phone.Extention'

Open strings

Open strings are defined without quotes.  In the next examples all values are defined as open strings.

productName = Diamond heart
quantity = 1
price = 248.90
ipo.comment = Valentine's day packaging.
shipDate = 2000-02-14

There are certain restrictions on usage of the open strings. The open string can’t start from dollar sign, single or double quote and it can’t have any special non-visible symbols that require escaping.

If the string starts with a quote then quoted string should be used instead.

Open string can’t start with a $ dollar sign because a dollar is used for aliases.

The restrictions listed above do not work for the open strings defined with double equality "==". For example, you can define open string starting with quote and dollar sign like in the following example.

Comment == "Lorem ipsum" dolor sit amet
Price == $10

Both types of open strings continue until the end of the line and ignore leading and trailing spaces.

Multiline Open Strings

Multiline open string is a simple way to add long strings to your Malina document. It uses indents to define multiline strings   For example:

LoremIpsum =
    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Pellentesque a lectus gravida, congue.
    Quisque mollis ut odio sed facilisis. Donec dictum ullamcorper lectus, ultrices convallis justo.
==

If the first line doesn’t have any symbols then it will be ignored and actually start from the next line.

The ending double equality is used to specify the end of the string. It can be omitted. In this case, the string will end based on indents and all trailing white space and new line symbols will be trimmed.

Folded open string

Sometimes strings are too long and thus hard to read and edit. In this case, the folded open string should be used. In the folded open string, newlines become spaces. The empty line is used to define a new line symbol. For example:

LoremIpsum ==
    Lorem ipsum dolor sit amet, consectetur adipiscing elit. 
    Pellentesque a lectus gravida, congue tellus quis, sagittis massa. 
    Aenean sed odio nec tellus vulputate vestibulum. Sed sed pulvinar 
    ex. 

    Quisque mollis ut odio sed facilisis. Donec dictum ullamcorper 
    lectus, ultrices convallis justo volutpat at. Cras nibh tellus, placerat 
    in pretium sit amet, tempor sit amet nibh. Nulla tempus molestie ligula 
    sed semper.
==

All open strings don’t support escape characters and string interpolation. If you need to add a multiline string with special symbols or string interpolation then multiline single-quoted string has to be used.

Multiline quoted string

Double-quoted multiline string can be used to define multiline strings. It works pretty straightforward. The first line of the string starts after opening double quote. The second and the next lines of the string are indented. The string ends with closing quote. For example:

LoremIpsum =" Lorem ipsum dolor sit amet, consectetur adipiscing elit. Pellentesque a congue.
    Quisque mollis ut odio sed facilisis. Donec dictum ullamcorper lectus, convallis volutpat at"

Single-quoted string allows usage of string interpolation and escape characters in the multiline string. At the same time it works as folded multiline string:

LoremIpsum = '\tLorem ipsum dolor $CustomerName consectetur adipiscing elit.     Pellentesque a lectus gravida, congue tellus quis, sagittis massa. 
    Aenean sed odio nec tellus vulputate vestibulum. Sed sed pulvinar 
    ex. \n
        Quisque mollis ut odio sed facilisis. Donec dictum ullamcorper 
    lectus, ultrices convallis justo volutpat at. Cras nibh tellus, placerat 
    in pretium sit amet, tempor sit amet nibh. Nulla tempus molestie ligula 
    sed semper.'
