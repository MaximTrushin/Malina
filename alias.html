Alias and Alias Definition

Alias definition

Malina has a very powerful code reuse feature called Alias. Basically, Alias is a short name for a fragment of the code. It can be used in place of element, attribute, block or value.

Aliases are defined in the module. Alias Definition starts with “!$” followed by alias name and block or value. For example:

!$Address.US:
    @xsi.type = "ipo:US-Address"
    ipo.name = Robert Smith
    ipo.street = 8 Oak Avenue
    ipo.city = Old Town
    ipo.state = NJ
    ipo.zip = 95819

!$Pi = 3.14159265359

Alias can be also defined with parameters. Parameters are defined inside the Alias Definition. Each parameter can be defined with the optional default value.

There are two types of parameters. Value Parameter represents a value (for example element’s or attribute’s value). Block Parameter represents a block (one or more attributes or elements).

The Value Parameter has the following syntax. %parameter_name [= optional_default_value]

The Block Parameter has the following syntax.

%parameter_name [: optional_default_block]

For example the following alias Items.Jewelry.Rings.Amber is defined with Value Parameter quantity with the default value 1.

!$Items.Jewelry.Rings.Amber:
    item:
        @partNum = 238-KK
        productName = Amber ring
        quantity = %quantity =  1
        price = 89.90
        ipo.comment =  With no inclusions, please.
        shipDate = 2000-01-07

In the next example alias Templates.PurchaseOrder is defined with Block Parameters ShipTo, BillTo, Items. Parameter ShipTo has a default block.

!$Templates.PurchaseOrder:
    ipo.purchaseOrder:
        shipTo:
            %ShipTo:                 
                ipo.name =  Helen Zoe
                ipo.street =  47 Eden Street
                ipo.city =  Cambridge
                ipo.postcode =  126
        billTo:
            %BillTo
        Items:
            %Items 

Alias

Alias can be used in place of any element or attribute if it is a Block Alias, or in place of value if it is a Value Alias.

Alias starts with $ following by alias name. For example the following document uses two Block Aliases $Address.EU and $Address.US 

!purchaseOrder:
    ipo.purchaseOrder:
        @orderDate = 2001-10-10
        shipTo:
            $Address.EU
        billTo:
            $Address.US

 

In the next example alias $States.Alaska is used to fill in the correct abbreviation for the state of Alaska.

billTo:
    ipo.name =  Robert Smith
    ipo.street =  8 Oak Avenue
    ipo.city =  Old Town
    ipo.state =  $States.Alaska
    ipo.zip =  95819

 

The alias could be specified with arguments if its definition contains parameters. If a parameter doesn’t have a default value then the corresponding argument must be added to the alias.

Arguments are added inside the alias block. 

Value argument has the following syntax. .argument_name = value

$Items.Jewelry.Necklaces.Lapis: 
    .quantity =  2

 

Block arguments has the following syntax .argument_name: block

In the following example the alias “Templates.PurchaseOrder” has three block arguments ShipTo, BillTo and Items

$Templates.PurchaseOrder:
    .ShipTo:
        @xsi.type = "ipo:EU-Address"  @export-code = "1"
        ipo.name = Helen Zoe
        ipo.street = 47 Eden Street
        ipo.city = Cambridge
        ipo.postcode = 126
    .BillTo:
        @xsi.type = "ipo:US-Address"
        ipo.name = Robert Smith
        ipo.street = 8 Oak Avenue
        ipo.city = Old Town
        ipo.state = NJ
        ipo.zip = 95819
    .Items:
        item:
            @partNum = 833-AA
            productName = Lapis necklace
            quantity = 1
            price = 99.95
            ipo.comment = Need this for the holidays!
            shipDate = 2013-12-12
