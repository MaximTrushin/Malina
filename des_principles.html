Design principles

  1. Malina uses indents to define document structure similarly to Python. Inline definitions are also supported.

  2. String, numeric, data, etc. constants can be defined without using opening and closing quotes. Quoted strings are used in special occasions like inline definitions, strings with special (escaped) symbols, interpolated strings.

  3. Malina supports basic primitives of both XML (like elements and attributes) and JSON (like collections and lists). It makes Malina semantically compatible with both languages.

  4. Malina supports XML namespaces on syntax level. It improves code readability and editing.

  5. Malina provides code reuse features similar to code reuse features of modern programming languages in a form of parameterized aliases.
