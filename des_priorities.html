Design priorities

  1. Readability and editability

    2. Malina achieves the higher people-friendliness by using clean, intuitive and people-friendly syntax.


  2. Semantic compatibility with XML and JSON

    3. Semantic compatibility means that Malina document can be converted to either XML or JSON formats.


  3. Powerful code reuse features

    4. Malina allows higher productivity by using rich code reuse features which are unique among all existing data-formats and markup-languages.


  4. Minimum set of syntax rules

    5. Malina can be used by people who have never worked with markup languages.
